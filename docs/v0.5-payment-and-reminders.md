# 我们俩的小田地 0.5：微信支付与提醒上线清单

更新日期：2026-07-16

## 当前结论

- 普通商业小程序可以做微信订阅提醒，但不能在后台静默获得永久“每天提醒”权限。用户必须主动点击授权；一次性订阅通常每授权一次可发送一条。
- 当前 0.5 已完成事项创建、提醒意愿、授权额度存储和手机日历兜底。微信消息发送尚未启用，因为公众平台的模板 ID 与字段名还没有配置。
- 个人主体小程序不能开通微信支付。正式收费前要把小程序主体准备为可认证的非个人主体、完成微信认证、申请并绑定微信支付商户号。
- “会员、主题、数字权益”属于虚拟服务，尤其涉及 iOS 时不能直接套用普通实物交易支付路径。上线前必须以微信公众平台和商户平台当时显示的虚拟支付准入规则为准。

## 微信提醒：还需要完成的步骤

1. 微信公众平台 → 功能 → 订阅消息 → 从公共模板库选择与“待办/日程提醒”匹配的模板。
2. 复制模板 ID，并记录每个字段的准确名称，例如 `thing1`、`time2`、`phrase3`；模板 ID 不是密码，可以交给开发者，AppSecret 不要发送。
3. 确认小程序 AppID 已和当前 CloudBase 环境关联。
4. 将模板 ID 和字段映射写入云函数环境变量，启用定时扫描与 `cloud.openapi.subscribeMessage.send`。
5. 用体验版真机测试：点击授权一次 → 创建一个几分钟后的事项 → 收到一条订阅消息 → 授权额度从 1 变为 0。

发送规则必须满足：只有发送成功才消耗额度；失败保留额度；同一事项幂等发送；模板字段严格限长；开发/体验/正式版使用正确的 `miniprogramState`。

官方参考：[CloudBase 订阅消息云函数指南](https://docs.cloudbase.net/recipes/add-subscribe-message-cloud-function)。

## 微信支付：资质与技术步骤

### 资质阶段

1. 在微信公众平台确认当前小程序“主体信息”。个人主体不能开通支付。
2. 使用符合平台要求的非个人主体完成微信认证。主体类型、经营范围和服务类目必须与实际产品一致。
3. 申请微信支付商户号，并在商户平台把商户号与当前小程序 AppID 绑定。
4. 在小程序后台开通所需支付产品，并确认数字会员/虚拟权益在 Android 与 iOS 的最新准入规则。

### 技术阶段

1. 在 CloudBase 集成中心创建“小程序微信支付”集成。
2. 只在集成中心填写商户号、APIv3 密钥、商户证书私钥、证书序列号、微信支付公钥及公钥 ID；不得写进 GitHub 或聊天。
3. 增加订单、退款和情侣会员权益集合。
4. 下单时由服务端生成唯一订单号与价格；前端不能决定实际金额。
5. 支付回调必须验签、核对金额并做幂等状态更新，只有服务端收到成功回调才发放权益。
6. 补齐取消支付、查单、退款、权益回收、客服和对账流程后，再把当前“会员内测”按钮切换为真实收费。

官方参考：[CloudBase 小程序微信支付集成指南](https://docs.cloudbase.net/integration/wechat-pay-miniprogram)。

## 发布闸门

在以下四项全部完成前，0.5 继续保持“会员体验/意向登记”，不会扣费：

- 主体与微信认证符合支付条件；
- 商户号已绑定 AppID；
- 虚拟权益类目与 iOS 规则已在当前控制台确认；
- 支付回调、退款和权益幂等测试全部通过。
