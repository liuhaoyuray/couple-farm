# 我们俩的小田地 0.9.0 发布素材

## 微信后台填写

- 版本号：`0.9.0`
- 版本说明：`修复跨用户头像和帖子图片，新增双人五子棋、断点续局与伴侣消息提醒。`

## 本次升级

- 修复头像和村庄帖子图片长期停在“照片正在加载”的问题。旧版直接从小程序端换取云存储临时地址，并把地址无限期缓存；私有读权限或地址过期后会反复走同一条失败路径。
- 新版图片请求先经过云函数确认查看者是本人、伴侣、同村成员或允许展示的社区成员，再签发一小时内有效的 HTTPS 地址；客户端只短期缓存。
- 若腾讯云临时地址在某台真机仍无法渲染，客户端会通过同一套服务端权限校验下载图片，并写入小程序私有目录显示。
- 两条图片路径都失败时会显示“图片暂时无法显示，点此重试”，不会无限显示加载状态。
- 新增 15×15 双人五子棋。发起者执黑，云端校验双方身份、回合、棋位和胜负，支持离开后继续、每 5 秒同步、手动刷新、认输和再来一局。
- 开局、轮到伴侣和胜负结果会进入情侣消息盒子；有可用的一次性微信订阅额度时沿用现有微信提醒能力。
- 部署流水线新增游戏数据集合健康检查，云函数、数据库和小程序构建全部通过后才上传体验版。

## 双真机验收清单

1. A 上传图片头像并保存，B 关闭并重新打开小程序，确认能看到 A 的图片头像。
2. A 在双方所在村庄发布一张新图片，B 打开村庄，确认图片在 12 秒内显示；再退出并重进，确认仍能显示。
3. 关闭网络后打开一张未缓存图片，应在超时后出现“点此重试”；恢复网络点击后应正常显示。
4. A 发起五子棋并落子，B 打开“相伴 → 一起玩”，确认棋盘同步且不能替 A 连续落子。
5. 双方交替完成横向、纵向或斜向五连，确认胜负一致；结束后由任一方开始新一局。
6. 检查消息盒子中存在开局、轮到落子和胜负消息。
7. 回归头像/帖子上传内容安全：违规内容只能提示“你发布的内容含违规信息。”，不能进入共享数据。

## 发布说明

GitHub Actions 会先部署 CloudBase 云函数和新集合，再上传微信小程序开发版本。上传成功后，管理员仍需在微信公众平台把 `0.9.0` 选为体验版；正式上线仍需走微信审核。

发布操作只在所有本地测试结束后统一进行。开发期间不再反复触发 GitHub 设备授权；临时工作区不能安全永久保存 OAuth 凭据，因此若发布时现有会话已失效，最多集中进行一次设备确认，凭据不会写入代码库。
