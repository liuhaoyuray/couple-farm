# 我们俩的小田地 0.5.0 发布素材

## 开发版本

- 版本号：`0.5.0`
- 版本说明：`新增共同小本本、共享事件与待办、手机日历和微信提醒接入，并加强村口连接。`

## 本版新增

- 全局品牌更新为“我们俩的小田地”，新用户默认昵称改为“田地新朋友”。
- 新增“共同小本本”底部入口：共享备忘、待办与事件。
- 支持类别、负责人、日期时间、每天/每周/每月重复和事项前一小时提醒意愿。
- 双方共同负责的待办需要两个人分别完成；指定负责人的待办只由负责人完成。
- 事项可加入手机系统日历；微信订阅提醒授权与额度存储已接入，待公众平台模板配置后启用发送。
- 首页增加“今天的小本本”和 24 小时内事项提醒，减少情侣间反复口头确认。
- 村口改为分区容错读取：帖子、留言、点赞、关注或排行榜中的某个查询异常时，页面仍可打开，并显示温和的降级提示。
- 社区查询移除对复合索引的非必要依赖，新增真实的“部分集合读取失败仍返回 200”自动化测试。
- 延续 0.4 的自适应如厕趋势、更多 Emoji 和图片头像能力。

## 体验版重点检查

1. 两名旧用户的配对、记录、动态和头像保持不变。
2. 点击“村口”能正常打开；即使暂无帖子也显示空状态而不是“暂时打了个盹”。
3. 一方新建共同事项，另一方立即可见；两个人完成共同待办后才进入“已完成”。
4. 指定给一方的事项，另一方不能代为完成。
5. 重复事项完成后只生成一条下一周期事项。
6. 有日期的事项能加入手机日历。
7. 未配置微信模板时，提醒入口明确说明“正在接入”，不会假装已经发送。
8. 新用户看到“田地新朋友”，可修改昵称、Emoji 或图片头像和田地名称。

## 审核说明补充

共同小本本里的备忘、待办和事件只向当前绑定的两名用户展示，不进入公开社区。微信订阅提醒必须由用户主动点击授权，未授权不发送。当前会员功能仍为免费体验和付费意向登记，不发起支付、不扣费。

自动流水线上传的是微信小程序“开发版本”。管理员仍需在微信公众平台把 `0.5.0` 选为体验版；正式上线前需更新隐私声明、服务类目并提交版本审核。
