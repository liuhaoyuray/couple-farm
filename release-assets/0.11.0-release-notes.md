# 我们俩的小田地 0.11.0 发布素材

## 微信后台填写

- 版本号：`0.11.0`
- 版本说明：`修复五子棋和井字棋对局，新增情侣杯计分、轮换先手与安静通知。`

## 本次升级

- 根治五子棋、井字棋首步落子触发 `GAME_INTERNAL_ERROR`：删除仅在测试替身中通过、在生产 CloudBase 运行时失败的条件批量更新路径，改用已验证的单文档更新。
- 保留双重 revision 校验、客户端动作 ID 和超时重试幂等；同一次操作即使重复到达，也不会重复记步或重复计分。
- `game-health` 会在真实云数据库创建一手测试棋、完成与正式对局相同结构的更新、读取验证后删除；发布只有通过真实写入才继续。
- 三款游戏增加持久化比分：双方胜场、平局、累计局数、当前连胜；游戏屋顶部汇总为“情侣杯”总比分。
- 五子棋和井字棋每局自动交换先手；胜负、平局和认输都计入比分。
- 开局、落子和出拳不再自动生成通知。整局结果只进入站内消息盒子，不发送微信订阅消息。
- 等待对方时可主动点击“提醒来玩”；同一游戏 30 分钟内最多提醒一次，避免打扰和浪费订阅额度。
- 新增结果卡片、胜利反馈、轻震提示、双方棋子角色和局内战绩，提升完整对局体验。

## 双真机验收清单

1. A 新开五子棋并落第一子，云端确认后 B 在约 3 秒内看到；双方连续完成至少 6 手，不出现诊断码或回滚。
2. A 新开井字棋，双方完成一局三连；比分应变为 1:0，并显示胜利结果卡。
3. 点击“再来一局”，上一局后手应成为新一局先手，累计比分保持不变。
4. 完成一局平局和一次认输，分别检查平局数、胜场和总局数只增加一次。
5. 猜拳双方选择前互相看不到手势，揭晓后胜负计入情侣杯。
6. 连续落子、出拳和开新局时消息盒子及微信均不应产生逐步通知。
7. 等待对方时手动点击“提醒来玩”，对方收到一条消息；30 分钟内重复点击应被温和拦截。
8. 断网落子后恢复网络并点击同步，棋盘恢复到云端状态；同一动作重试不能重复计步。

## 发布说明

GitHub Actions 会先运行双账号完整对局测试，再部署 CloudBase 云函数；线上 `game-health` 必须完成真实数据库写入验证，之后才发布大陆 H5。微信小程序 `0.11.0` 上传成功后，管理员仍需在公众平台将最新开发版本选为体验版。
