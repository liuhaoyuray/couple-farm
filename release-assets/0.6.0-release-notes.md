# 我们俩的小田地 0.6.0 发布素材

## 微信后台填写

- 版本号：`0.6.0`
- 版本说明：`新增个人体验田、熟人村庄、一级趋势入口、日程地点与微信订阅提醒。`

## 本次升级

- 新用户完成昵称设置后直接进入个人体验田，不再被配对页阻断。
- 未配对用户可以记录、编辑和删除自己的体重与如厕数据，并查看 7/30/90 天体重曲线和 7 天如厕趋势。
- 配对成功时，双方个人体验田中的历史记录自动迁入共同田地。
- 原公开“村口”替换为邀请制熟人村庄：一对情侣创建，其他认识的情侣使用 8 位邀请码加入。
- 村庄包含成员名册、村长资料管理、30 天邀请码、换码、文字/单图动态、每日话题、点赞、留言、举报、退出、村长交接和解散。
- 体重和如厕原始记录永远不会进入村庄；村庄内容仍执行微信文字与图片安全检查。
- “趋势”恢复为底部一级入口；如厕柱高使用对数缩放，单日次数较多时也不会撑坏图表。
- 共同小本本新增地点，并接入微信“日程提醒”模板。提醒字段为内容、开始时间、地点和发布人。
- 微信订阅消息一次授权只对应一次发送额度；手机系统日历继续作为长期提醒兜底。
- 支付链路本版本保持关闭，待小程序正式发布并完成商户资质后再接入。

## 体验版验收清单

1. 用一个从未配对的微信打开小程序，设置昵称后应直接进入“体验田”，可记体重与如厕。
2. 在底部点“趋势”，确认体重曲线和如厕趋势均可见。
3. 两个账号各自先记一条数据，再使用配对码绑定，确认两边旧记录都保留。
4. 第一对情侣创建村庄，把邀请码发给第二对情侣；第二对加入后发帖，第一对应能看到、点赞和留言。
5. 创建带地点和未来时间的共同事项，开启微信提醒并在订阅弹窗点“允许”。
6. 到提醒时间后核对服务通知；若定时触发链路暂时无微信调用凭证，任一方打开小程序会触发补发检查。
7. 解除绑定前先确认村庄关系、共同记录和个人记录的展示符合预期。

## 发布说明

自动流水线上传的是微信小程序“开发版本”。上传成功后，管理员仍需在微信公众平台将 `0.6.0` 设为体验版；正式上线前需完成隐私声明、服务类目、内容安全说明和版本审核。
