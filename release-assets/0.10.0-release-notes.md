# 我们俩的小田地 0.10.0 发布素材

## 微信后台填写

- 版本号：`0.10.0`
- 版本说明：`修复五子棋卡顿和落子失败，新增井字棋、默契猜拳与断线自动同步。`

## 本次升级

- 五子棋棋盘由 225 项数组改成紧凑状态串，同时兼容读取 0.9.0 已创建的旧棋局。
- 每一步带客户端动作 ID 和棋盘版本；请求超时后使用同一 ID 重试，不会重复落子。
- 手机先乐观显示棋子，再等待云端确认。轮到对方时约 3 秒同步，轮到自己时降低轮询频率；同一时刻最多保留一个同步请求。
- 游戏进行中的微信通知进入五分钟云端队列，不再让订阅消息发送阻塞落子请求。
- 错误从短 Toast 改成常驻同步提示，完整展示并可复制诊断码。
- 新增井字棋：3×3 棋盘、回合校验、三连胜负、平局、认输、断线续局和再来一局。
- 新增默契猜拳：石头/剪刀/布选择在双方都提交前保密，提交后同时揭晓并由服务端判断胜负。
- 游戏健康检查现在声明并验证 `gomoku`、`tic-tac-toe` 和 `rps` 三项能力。

## 双真机验收清单

1. A 打开旧的五子棋第 2 局并在空位落子，棋子应立即显示，随后提示轮到 B；不能再出现无限等待或短 Toast 截断诊断码。
2. B 保持游戏页打开，A 落子后约 3 秒内 B 自动看到；双方快速点击时每回合仍只能成功一手。
3. A 落子后立刻切走小程序，再返回游戏屋，棋盘和回合应与 B 一致。
4. 关闭网络后尝试落子，应在约 8 秒内显示可复制的错误信息；恢复网络点击“同步”后棋局继续。
5. 双方完成一局五子棋的五连、认输和再来一局。
6. 双方完成一局井字棋的三连和平局，并验证不能连续替对方落子。
7. 猜拳时 A 先选，B 只能看到“对方已出拳”而看不到手势；B 选择后两台手机同时看到相同结果。
8. 回归头像和帖子图片、体重/如厕趋势、村庄、消息盒子及内容安全发布流程。

## 发布说明

GitHub Actions 会先部署 0.10.0 CloudBase 云函数并执行三项游戏健康检查，再上传微信小程序开发版本。上传成功后，管理员需在微信公众平台把 `0.10.0` 选为体验版；正式上线仍需走微信审核。
